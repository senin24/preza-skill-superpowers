Проблема

Агент без инструкций — это лотерея

Без скилла

  • «Глупая» текущая модель: агент часто плохо угадывает, что вы имели в виду
  • Сразу пишет код — без глубокого анализа контекста
  • Небольшое контекстное окно: Агент быстро расходует лимит токенов и падает
  • Нет плана работы — итог непредсказуем

С навыками Spec Driven Development

  • Сначала изучает проект и контекст
  • Находит аналоги в кодовой базе
  • Строит подробный план из атомарных небольших шагов
  • Экономит контекст при имплементации
  • Результат часто предсказуем и легко проверяем

Скилл requesting-code-review

Код-ревью через субагент-ревьюер

Свежий субагент-ревьюер получает только код и контекст задачи — никакой истории сессии. Независимый взгляд на результат.
1
Вызов: /skills requesting-code-review → агент берёт BASE_SHA и HEAD_SHA из git log
2
Диспетчеризирует code-reviewer субагент: что сделано, план, диапазон коммитов
3
Субагент читает diff, запускает mvn test, проверяет соответствие плану
4
Возвращает: Strengths / Issues (Critical → Important → Minor) / Overall Assessment

Текущая ситуация

Текущие преграды для буста агентской разработки

  • Нет YOLO-режима (You Only Live Once) *
  • Медленная кодерская LLM (ждем железо)
  • Небольшое контекстное окно в 130k токенов (обещают 200k)
  • Неадекватный Цензор (он вообще нужен?)
  • Нет мощной аналитической LLM (желательно другого вендора)
  • Относительно высокий порог вхождения для не тех персонала
  • Всё в целом работает медленно и с «танцами с бубном»
  • Много времени уходит на эксперименты и исследования
  • Методология работы в зачаточном состоянии
