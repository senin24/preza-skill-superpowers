== SLIDE ==
title: Агент без инструкций — это лотерея
tag: Проблема
layout: two-col

[LEFT: Без скилла | bad]
- «Глупая» текущая модель: агент часто плохо угадывает, что вы имели в виду
- Сразу пишет код — без глубокого анализа контекста
- Небольшое контекстное окно: Агент быстро расходует лимит токенов и падает
- Нет плана работы — итог непредсказуем

[RIGHT: С навыками Spec Driven Development | good]
- Сначала изучает проект и контекст
- Находит аналоги в кодовой базе
- Строит подробный план из атомарных небольших шагов
- Экономит контекст при имплементации
- Результат часто предсказуем и легко проверяем

== SLIDE ==
title: Superpowers — жесткая дисциплина разработки
tag: Расширение · github.com/obra/superpowers
layout: plan

- Изучение проекта и зависимостей
- Предварительный анализ задачи в контексте проекта
- Уточняющие вопросы
- **Генерация спеки** (системное требование) для задачи
- **Генерация плана** работ по системному требованию
- **Пошаговая имплементация** плана через субагенты
- **Верификация** результата (сборка, тесты)
- Ревью субагентом на каждом шаге

== SLIDE ==
title: Библиотека скиллов Superpowers
tag: Что внутри
layout: plan

- **brainstorming** — уточняющие вопросы, 2–3 подхода, утверждение дизайна до строчки кода
- **writing-plans** — разбивка на задачи по 2–5 минут с точными путями файлов и командами
- **subagent-driven-development** — свежий субагент на каждую задачу, двухэтапное ревью
- **requesting-code-review** — независимый субагент-ревьюер: diff + тесты + оценка
- **red-green-refactor** — обязательный TDD: сначала падающий тест, потом минимальный код
- **systematic-debugging** — 4-фазный анализ первопричины вместо random-правок
- **verification-before-completion** — объективная проверка до отчёта "готово"
- **writing-skills** — создание собственных скиллов под нужды команды

== SLIDE ==
title: Код-ревью через субагент-ревьюер
tag: Скилл requesting-code-review
layout: steps

highlight: Свежий субагент-ревьюер получает только код и контекст задачи — никакой истории сессии. Независимый взгляд на результат.

1. Вызов: `/skills requesting-code-review` → агент берёт BASE_SHA и HEAD_SHA из `git log`
2. Диспетчеризирует `code-reviewer` субагент: что сделано, план, диапазон коммитов
3. Субагент читает diff, запускает `mvn test`, проверяет соответствие плану
4. Возвращает: Strengths / Issues (Critical → Important → Minor) / Overall Assessment

== SLIDE ==
title: Текущие преграды для буста агентской разработки
tag: Текущая ситуация
layout: case

- Нет YOLO-режима (You Only Live Once) *
- Медленная кодерская LLM (ждем железо)
- Небольшое контекстное окно в 130k токенов (обещают 200k)
- Неадекватный Цензор (он вообще нужен?)
- Нет мощной аналитической LLM (желательно другого вендора)
- Относительно высокий порог вхождения для не тех персонала
- Всё в целом работает медленно и с «танцами с бубном»
- Много времени уходит на эксперименты и исследования
- Методология работы в зачаточном состоянии
